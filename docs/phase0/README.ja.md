# Phase 0: AI駆動開発のためのコンテキスト設計

## 目的
Phase 0 は、実装前に「AIが正しく判断するための前提情報」を統一し、以後の開発フェーズでの手戻りを減らすための準備フェーズです。

- 実装の前に、判断基準・語彙・入出力契約を固定する
- 人間向け仕様とAI向け仕様を分離し、更新責務を明確化する
- スコープ外の議論を減らし、Phase 1 以降を実装中心にする

## 言語ポリシー（共通コンテキスト）
- Chat: 日本語
- AI関連ドキュメント: 英語
- 人間向けドキュメント: 日本語
- Code / comments / naming: 英語

## Phase 0 の成果物
1. `docs/phase0/ai_context_contract.md`
   - AIが参照する不変条件（スキーマ、評価軸、失敗時ルール）
2. `docs/phase0/context_map.ja.md`
   - フェーズ別・タスク別で、どの情報を参照すべきかの索引
3. `docs/phase0/decision_log.ja.md`
   - 仕様上の意思決定ログ（なぜその設計にしたか）

## 完了条件（Definition of Done）
- 評価軸、分類ルール、JSON出力契約が単一文書に統合されている
- AI向け/人間向け文書の役割分担が明文化されている
- Phase 1〜6 で参照する文書がタスク単位で紐づいている
- 後続フェーズの開始時に「追加確認が必要な未定義項目」が一覧化されている

## 運用ルール
- 仕様変更時は、まず Phase 0 文書を更新してから実装を変更する
- PR には「どのコンテキスト契約を更新したか」を必ず記載する
- AIプロンプト変更は、必ずスキーマ更新と同時にレビューする
